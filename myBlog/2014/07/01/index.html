The Personal Website Battle Continues

After some careful reading of the directions, a few hours of missteps and some coffee. I successfully used Github Pages to host my Fargo blog at kevinraymond.net.

A few things I learned....

  1. In Github, committing is kinda like publishing. Once you have rendered the pages in Fargo, they are added to the folder in your Dropbox ( if you have added, #siteFolder "username.github.io" in your outline. It's in the directions.) and ready to be committed in the Github program on your Mac or Windows machine.

  2. I use a Chromebook, but have a Windows computer at work which I can log into anytime I need. Therefore, I can render my pages and go to my Windows machine remotely to commit them in the Github program. I guess this is what you would call a work around.

  3. Reading directions is key. So read them here.

  4. Patience is your friend. One of the main reasons I got so frustrated was my lack of patience for things to reconcile on the server etc. Waiting 10 minutes was like an eternity. I blame society for this. I think the best pet detective in the biz, Ace Ventura said it best, "If I'm not back in five minutes. Just wait longer."

The excitement of the accomplishment was euphoric. Strange for something so simple. I have basic computing skills, but this was my first venture into something this tricky. And honestly, it wasn't difficult after taking the time to check all my steps.

However.....

Now I am wondering if hosting a regular site at kevinraymond.net is better and using something like blog.kevinraymond.net for the blog, which would be hosted through Github. I have no idea if this makes sense or if I am being a non savvy computer person. ( Apologies for lack of terminology if you're reading this and live in the computing world. I fabricate sheet metal for a living so I am kind of a layman)

Anyway. My thought is, I could host the main site, giving me the ability to create sub pages like blog.kevinraymond.net and links.kevinraymond.net for my Fargo blog and link page.

Am I on the right track? I realize there are so many ways to get a site started. I am hoping to keep it low cost and use my brain to figure some things out. Plus, it's fun.

If you have any suggestions please leave them in the comments or tweet me @KevRaymond. Thanks!
